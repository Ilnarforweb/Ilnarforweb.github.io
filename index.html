Язык javascript является однопоточным.

Это не лучшим образом сказывается, в частности, на работе UI.

Выходом является асинхронное выполнение программы. Например с помощью функции:


						setTimeout(callback, milliseconds);

setTimeout(callback, milliseconds)

Функция позволяет планировать выполнение событий , которые должны произойти позже момента обращения к этой функции


					console.log('Hi');
					setTimeout(function cb1() { 
					  console.log('cb1');
					}, 5000);
					console.log('Bye');

Стандарт HTML5 превнёс возможность использовать WEB WORKER.

Веб-воркеры не являются частью JavaScript.

Веб-воркеры — это потоки, принадлежащие браузеру, которые можно использовать для выполнения JS-кода без блокировки цикла событий.

Создание веб-воркера:

var worker = new Worker('task.js');

Запуск созданного воркера:

worker.postMessage();

Остановка воркера со страницы сайта:

 worker.terminate()

Остановка изнутри:

 self.close()

Возможности, доступные веб-воркерам:

  • Объект navigator
  • Объект location (только для чтения)
  • XMLHttpRequest
  • setTimeout()/clearTimeout() и setInterval()/clearInterval()
  • Кеш приложения
  • Импорт внешних скриптов с использованием importScripts()
  • Создание других веб-воркеров

Для веб-воркера не доступны:

  • DOM структура основной страницы
  • Объект window
  • Объект document
  • Объект parent

Таким образом веб-воркер ориентирован на сложные вычисления, а обработка UI происходит отдельно.
